# Vision Agent 产品文档

> 更新日期：2026-07-15  
> **一句话**：拍一下，让合适的 AI 专家帮你看懂眼前的世界。

## 是什么

以**相机为第一入口**的视觉智能体：拍照/选图 → AI 解读 → 可追问、可回看、可分享。

| 端 | 仓库 |
|----|------|
| App | `vision-agent-mobile`（Expo / RN） |
| API | `vision-agent-api`（Node / Fastify） |

线上 API：`https://vision-agent-api.onrender.com`

## 用户怎么用

```text
打开相机 →（可选）选镜头 → 拍照/相册 → 看洞察 → 追问 / 分享 / 回看记忆
```

| 镜头 | 做什么 |
|------|--------|
| 自动 | 系统选专家 |
| 食识拍 | 营养估算报告（需**手动选**，自动模式不会当成食识拍） |
| 美食探索 | 风味、文化、周边 |
| 穿搭 / 本地向导 / 艺术 / 设计 / 文字 / 好奇心 | 对应专项解读 |

## 两条分析策略

| 入口 | 目标 | 做法 |
|------|------|------|
| **手动专项**（如食识拍） | 字段完整 | 完整看图 + 深度推理，不砍结构 |
| **自动** | 更快 | 轻量看图 → 路由 → 视觉直出洞察 |

原则：专项体验不能为了速度牺牲关键字段。

## 体验要点

- 分析要有进度反馈；失败要说清楚原因  
- 「附近/周边」追问：先说明再要定位；已授权则静默取位置  
- 营养等结论均为**估算**，非医疗/过敏诊断  
- Render 免费档可能冷启动，先打 `/health` 预热  

## 主要接口

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/health` | 探活 |
| GET | `/v1/agents` | Agent 列表 |
| POST | `/v1/analyze` | 分析（可 `agent_override`） |
| POST | `/v1/analyze/stream` | 流式分析 |
| POST | `/v1/followup` | 追问（可带 lat/lng） |
| GET/DELETE | `/v1/memories`… | 历史记忆 |
