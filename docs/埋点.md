# Vision Agent 埋点

> PostHog · 仅移动端 · 技术实现：`src/services/analytics.ts`

## 先记住这 3 点

1. **别看 Web analytics**（那是给网站的）。App 用 **Product analytics**。  
2. App 没有网址 path，有的是 **`$screen`（屏名）** + 业务事件。  
3. 地图空、没 path ≠ 埋点失败；事件进 **Activity → Live** 就算通了。

## 配置

```bash
# .env （改完重启 Metro）
EXPO_PUBLIC_POSTHOG_API_KEY=phc_xxxxxxxx
EXPO_PUBLIC_POSTHOG_HOST=https://us.i.posthog.com   # 欧区用 eu.i.posthog.com
```

无 key 时不上报。正式包请在 `eas.json` 的 env 里填 key（勿把真实 key 提交 git）。

## 怎么看数据

| 想看什么 | 去哪 |
|----------|------|
| 事件是否到了 | Activity → Live / Events |
| 主路径流失 | Insights → **Funnel** |
| 屏幕跳转 | Insights → **Paths**（事件选 `$screen`） |
| 趋势 / 分 Agent | Insights → **Trends** |
| 日常固定看 | 自建 Dashboard「App 看板」 |

**主漏斗：**  
`app_open` → `analyze_start` → `analyze_success` → `insight_view` → `followup_ask`

**常用比率：**

- 分析完成率 = `analyze_success` / `analyze_start`  
- 追问率 = `followup_ask` / `insight_view`  
- 分享率 = `share_poster` / `insight_view`

**城市 / 「上海」：**  
看事件属性 `$geoip_city_name`（按 IP，不是 App GPS）。不要用 Web analytics 的 Geography Map；要用 Product Insight 按 Country/City 拆。

## 事件一览

| 事件 | 何时 | 关键属性 |
|------|------|----------|
| `app_open` | 冷启动 | — |
| `$screen` | 切屏 | `$screen_name`：`Camera` / `Home` / `Insight`… |
| `camera_mode_select` | 切换镜头 | `mode_id`, `agent` |
| `analyze_start` | 开始分析 | `agent`, `source`：`camera` \| `gallery` |
| `analyze_success` / `fail` | 分析结束 | `duration_ms`, `has_location` / `error` |
| `insight_view` | 进洞察页 | `entry_mode`：`fresh` \| `history` |
| `followup_ask` | 提交追问 | `source`：`chip` \| `input` |
| `location_prompt` | 附近追问要定位 | `result`：`deny_prompt` / `accept_prompt` / `grant` / `deny_system` |
| `memory_open` / `delete` | 打开 / 删除记忆 | `from`：`home` \| `profile` |
| `share_poster` | 分享成功 | `agent` |

**不上报：** 追问原文、图片 URL、经纬度数值、账号信息。

## 代码约定

```ts
import { track } from '@/services/analytics';
track('event_name', { agent: agentId });
```

- 只通过 `track` / `trackScreen`，不要自己 `new PostHog()`  
- 事件名 `snake_case`；属性用稳定枚举  
- 接线：`App.tsx` + `NavigationAnalytics.tsx`（RN v7 手动上报屏幕）

## 验证

配好 key → 重启 App → 走一遍拍照/追问 → Live 里看到事件即可。
